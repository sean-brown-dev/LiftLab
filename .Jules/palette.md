## 2025-04-06 - Replacing null contentDescriptions with meaningful strings in Compose
**Learning:** Found multiple interactive `Icon` composables in the application using `contentDescription = null`. This degrades accessibility significantly as screen readers cannot announce the purpose of these elements. Adding contextual string resources (e.g., `accessibility_close`, `accessibility_skip_timer`) using Jetpack Compose's `stringResource` provides necessary context.
**Action:** When creating or modifying Compose UI elements, always provide meaningful, localized `contentDescription` attributes via `stringResource` instead of `null` for interactive controls (like buttons, dropdown toggles, and actionable icons). Ensure newly added string resources are defined in `strings.xml`.
